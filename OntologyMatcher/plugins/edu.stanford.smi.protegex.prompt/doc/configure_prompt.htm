Configuring Prompt

You can configure PROMPT's behavior using the PROMPT Configuration dialog. 

To change PROMPT configuration:

  1. Select Configure from the Prompt menu. This menu appears to the right of the other menus when you enable PROMPT.

  2. To change the way PROMPT handles slots when you copy or merge a class, check Consider inherited slots. In general, classes have two kinds of slots: slots that are directly attached to a class and inherited slots. If Consider inherited slots is not selected, then each time you copy or merge a class, PROMPT only copies direct slots. If Consider inherited slots is selected, PROMPT also copies any inherited slots.

  3. To change whether or not frame names that are the same except for capitalization should be considered to be identical, check Case sensitive ontology. If this item is selected, frames that differ only in their capitalization are considered to be different. If this item is deselected, frames that have the same spelling but different capitalization are considered to have the same name. In this case, frames with the "same" name may show up at the Conflicts tab. 

  4.  When merging ontologies, to set one of the source ontologies as "preferred" for names, documentation strings, or roles, click Set preferred ontology and select the ontology you want from the menu. Normally, when you are merging two frames with different names, documentation strings, or roles (that is, different own slot values), PROMPT asks you to choose the value for the merged class. Setting an ontology as "preferred" tells PROMPT to automatically choose the name, documentation, and role of the frame that comes from the preferred ontology. Note that the ontology that is selected at the time that the operation is performed is the one that is used. This allows you to choose a different preferred ontology for different parts of the merge. For example, suppose you had two patient database ontologies, one of which was better structured for personal and billing information, and the other of which was better structured for medical history and medications. You could set the first one as the default and incorporate the personal information, then change the default to the second, without changing the frames you had already incorporated, and then continue your merge, incorporating the medical history.

Note: PROMPT configuration can only take place after you have enabled PROMPT and chosen your source ontologies. To learn about enabling PROMPT by selecting Configure from the Project menu, see Enabling the Prompt Tab.
